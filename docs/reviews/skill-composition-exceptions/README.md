# Skill Composition Exceptions

This directory holds explicit review records for skills that intentionally
remain single-technique bundles.

Use this surface when a skill is not a multi-technique package, but still
deserves a skill boundary because it adds a real workflow layer.

Required headings for each review:

- `Current shape`
- `Package rationale`
- `Why this is not just the technique`
- `Adjacent skills considered`
- `Recommendation`

Allowed recommendation values:

- `keep_exception`
- `reframe_multi_technique`
- `merge_or_remove`

The review should stay public-safe, bounded, and specific about why the skill
is still a skill rather than a thin technique mirror.
