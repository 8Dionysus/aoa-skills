# Review Checklist

## Purpose

Use this checklist when reviewing a bounded `abyss-*` runtime diagnosis change
that claims to follow `abyss-self-diagnostic-spine`.

## When it applies

- the change adds or updates one runtime-owned diagnosis artifact inside an `abyss-*` repo
- the review needs to confirm repo-relative evidence refs, exit posture, and owner-aware handoff stayed explicit
- the family review doc and the local bundle still need to stay aligned

## Review checklist

- [ ] The target path being diagnosed is concrete before verdicts are made.
- [ ] Runtime evidence refs stay repo-relative, cited, and public-safe.
- [ ] Drift classes and unknowns are named explicitly instead of being flattened into one collapse verdict.
- [ ] One exit class is chosen and it stays smaller than repair authority.
- [ ] Any handoff stays pointed at the right owner layer and does not absorb session, repair, progression, or quest canon.
- [ ] The final note stays a thin local adaptation of `aoa-session-self-diagnose`, not a new project doctrine.

## Not a fit

- requests for immediate silent repair or free self-mutation
- cases where the base `aoa-session-self-diagnose` skill is already sufficient without `abyss-*` local adaptation
