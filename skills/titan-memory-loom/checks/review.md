# Review Checklist

## Purpose

Use this checklist when reviewing `titan-memory-loom` as part of a Titan service-cohort session.

## When it applies

- the skill output affects a Titan receipt, bridge ledger, console state, approval record, or memory candidate
- Forge mutation or Delta judgment posture may be involved
- the result needs to stay below owner-repo truth and human judgment

## Review checklist

- [ ] The operator-visible intent is explicit.
- [ ] The target Titan lane and gate kind are named when relevant.
- [ ] Forge mutation and Delta judgment gates remain distinct.
- [ ] Receipt, ledger, console, replay, or memory refs are preserved when available.
- [ ] Candidate memory is not promoted to owner truth.
- [ ] The output names validation, replay, or owner-route follow-up when needed.
- [ ] No hidden background execution or auto-approval was introduced.

## Not a fit

- requests that infer approval from silence
- hidden execution or silent mutation
- memory canonization without owner confirmation
- proof or verdict claims wider than the bounded evidence
